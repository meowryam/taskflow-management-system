# Responsible AI Policy

This project follows responsible AI-assisted software engineering practices.

## Principles

- AI assists development but does not replace developer understanding.
- All AI-generated code is manually reviewed.
- Manual testing is required before merging.
- Sensitive information is never shared with AI tools.
- Git history accurately reflects the work performed.

## Allowed Uses

- Code generation
- Refactoring
- Documentation
- Debugging
- Testing assistance

## Not Allowed

- Fake commits
- Fake pull requests
- Fake reviews
- Fabricated reports
- Committing secrets or credentials

Every contributor is responsible for understanding the code they submit.
