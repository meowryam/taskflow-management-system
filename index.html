Responsible AI — Developer Guidance

  • Never share passwords, API keys, credentials, or private company information with AI tools.
  • Never commit .env files or secrets to version control.
  • Do not blindly trust AI-generated code — always verify logic and security.
  • Manually review and test all AI-generated output before using it in production.
  • Do not use AI to create fake commits, reviews, reports, or evidence of work.
  • Developers must understand and be able to explain all code they submit.
  • AI should assist with planning, debugging, testing, refactoring, and documentation — but must not replace human verification.
