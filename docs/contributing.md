# Contributing Guide

## Workflow

1. Create an Issue
2. Create a Branch
3. Commit changes
4. Open Pull Request
5. Request Review
6. Merge
7. Close Issue

## Branch Naming

feature/module-name

fix/bug-name

docs/document-name

release/version

## Commit Style

feat:

fix:

docs:

style:

refactor:

test:

chore:

## Pull Requests

Every PR should include:

- Description
- Linked Issue
- Testing Steps
- Screenshots (if applicable)
